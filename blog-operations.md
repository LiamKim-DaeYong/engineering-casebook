# 블로그 운영 원칙

## 목적

이 문서는 이 저장소의 학습 기록을 기술 블로그 글로 승격할 때 따르는 운영 원칙을 정리한다.

핵심은 학습 기록과 발행 글의 역할을 분리하는 것이다.
이 저장소의 `notes/`는 학습 원본을 쌓는 공간이고, 블로그는 충분히 숙성된 내용을 발행하는 공간이다.

## 기본 방향

- `notes/`는 학습 중인 내용을 빠르게 쌓는 원본 저장소로 유지한다.
- 블로그는 원본 메모를 그대로 옮기는 공간이 아니라, 다시 설명 가능한 수준으로 숙성된 결과물을 올리는 공간으로 본다.
- 블로그 발행 병목을 줄이기 위해 완성 글 작성보다 초안 생성과 검토 흐름을 우선 설계한다.
- 자동화의 목표는 자동 발행이 아니라 자동 초안 생성이다.
- 최종 발행은 항상 사람 검토와 승인 이후에 진행한다.

## Repo와 블로그의 역할 분리

### Repo

- 학습 메모
- 개념 정리 초안
- 실험 기록
- 비교 메모
- 아직 결론이 닫히지 않은 문서

### Blog

- 주제가 어느 정도 숙성된 글
- 독자가 읽어도 흐름이 이해되는 글
- 설명 순서와 제목, 요약, 예제가 정리된 글
- 검색 유입이나 재참조 가치가 있는 글

즉, repo는 연구 노트에 가깝고, 블로그는 선별된 출판본에 가깝다.

## 기존 글 처리 원칙

- 기존 블로그 글은 전면 수정하지 않는다.
- 기존 블로그 글은 전면 아카이빙하지 않는다.
- 기존 글은 현재 시점의 판단 기준으로 모두 다시 다듬지 않는다.
- 신규 글부터 새 운영 기준을 적용한다.

기존 글 중에서 현재 방향과 너무 맞지 않거나 대표성이 낮은 글이 있다면, 그때 선택적으로 비노출 또는 아카이빙 여부를 판단한다.
하지만 기본 방침은 유지다.

## 신규 글 운영 원칙

- 신규 글은 학습 중간 산출물이 아니라, 어느 정도 이해가 쌓인 뒤 승격한다.
- 신규 글은 `notes/`의 내용을 기반으로 하되, 블로그 독자가 읽을 수 있는 구조로 다시 정리한다.
- 신규 글은 기존 블로그의 설명형 톤을 유지하되, 그 규칙은 별도 블로그 프로필 문서에서 관리한다.
- 신규 글은 구조와 품질 기준을 일관되게 적용한다.

## 승격 기준

아래 조건 중 여러 개를 만족하면 블로그 초안 생성 후보로 본다.

- 주제를 내 문장으로 끝까지 설명할 수 있다.
- 문서의 범위와 핵심이 어느 정도 정리되어 있다.
- 실무, 설계, 운영, CS 관점에서 재사용 가치가 있다.
- 예제, 흐름, 비교, 문제 해결 과정 중 하나 이상이 포함되어 있다.
- 나중에 다시 읽어도 별도 맥락 없이 이해할 수 있다.

반대로 아래 상태라면 아직 `notes/` 단계에 머무는 편이 낫다.

- 개념이 아직 헷갈린다.
- 문서가 단순 발췌 수준이다.
- 핵심 질문은 있는데 답이 정리되지 않았다.
- 설명 순서보다 참고 자료 조각이 더 많다.

## 시리즈와 단편 판단 원칙

시리즈 여부는 `notes/` 작성 시점에 고정하지 않는다.
학습 중에는 단편처럼 보이던 주제가 나중에 시리즈로 확장될 수 있고, 반대로 시리즈 후보처럼 보이던 주제가 한 편으로 정리되는 경우도 있기 때문이다.

시리즈/단편 판단은 초안 승격 시점 또는 발행 직전 편집 시점에 한다.

판단 기준은 아래와 같다.

- 같은 상위 주제 아래 문서가 연속적으로 누적되는가
- 각 글이 독립적으로 읽혀도 이해 가능한가
- 한 글 안에 다루는 축이 너무 많아지는가
- 제목을 시리즈 번호 없이 붙였을 때 오히려 어색한가

원칙은 다음과 같다.

- 기본값은 단편이다.
- 주제 누적과 연결성이 명확할 때만 시리즈로 승격한다.
- 시리즈를 위해 억지로 분리하지 않는다.

## 자동화 원칙

- 자동화는 발행을 대신하지 않는다.
- 자동화는 초안 생성을 돕는 보조 장치로 사용한다.
- 초안은 규칙 기반으로 반복 생성 또는 갱신할 수 있다.
- 초안 생성 후에는 반드시 사람 검토를 거친다.
- 승인되지 않은 초안은 발행본으로 취급하지 않는다.
- 발행이 끝난 초안은 기본적으로 삭제하지 않고 보관한다.

## 승인 흐름

기본 흐름은 다음과 같다.

1. `notes/`에 학습 내용을 축적한다.
2. 승격 기준을 만족하는 문서를 블로그 초안 후보로 본다.
3. 별도 초안 공간에 블로그 초안을 생성한다.
4. 초안을 검토하며 구조, 설명 순서, 제목, 요약을 다듬는다.
5. 승인된 초안만 실제 블로그 발행 대상으로 넘긴다.

## 문서 분리 원칙

이 문서는 운영 정책만 다룬다.

아래 항목은 별도 문서에서 관리한다.

- 블로그 톤과 글쓰기 스타일
- frontmatter 규칙
- 카테고리/태그 규칙
- 초안 저장 위치와 상태 디렉토리
- 초안 생성 템플릿

즉, 이 문서는 "어떻게 운영할 것인가"를 다루고, 세부 실행 규칙은 하위 문서로 분리한다.

## 참고 방식

- 블로그 관련 작업을 할 때 이 문서를 상위 기준으로 삼는다.
- 세부 초안 생성 규칙이 필요하면 별도 `blog-profile.md`, `blog-pipeline.md` 같은 문서로 확장한다.
- `AGENTS.md`에는 필요할 경우 이 문서를 참조하라는 최소 연결만 추가한다.
