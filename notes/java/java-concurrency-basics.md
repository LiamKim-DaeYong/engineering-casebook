# Java 동시성 기초

## 개요

동시성 프로그래밍을 이해하려면 운영체제가 프로그램, 프로세스, 스레드를 어떻게 다루는지 먼저 정리해야 한다.
자바에서 `Thread`, `synchronized`, 스레드 풀을 다루는 것도 결국 운영체제의 실행 단위와 스케줄링 방식 위에서 동작한다.

이 문서는 멀티 태스킹, 프로세스, 스레드, 스케줄링, 컨텍스트 스위칭의 기본 개념을 정리한다.

## 멀티 태스킹

멀티 태스킹은 운영체제가 여러 작업을 짧은 시간 단위로 번갈아 실행해서 동시에 실행되는 것처럼 보이게 하는 방식이다.

예를 들어 음악 프로그램과 문서 프로그램을 함께 실행한다고 생각해보자.
음악 프로그램이 완전히 끝난 뒤에야 문서 프로그램을 사용할 수 있다면 사용자는 매우 불편할 것이다.
운영체제는 두 프로그램의 실행 시간을 잘게 나누고 빠르게 전환하면서 처리한다.
그 결과 사용자는 음악을 들으면서 동시에 문서를 작성하는 것처럼 느낀다.

이처럼 시간을 나누어 여러 작업을 처리하는 방식을 시분할 기법이라고 하며, 영어로는 time sharing이라고 한다.
애니메이션이 여러 장의 이미지를 빠르게 보여주면서 움직이는 것처럼 보이게 하는 것과 비슷하게, 운영체제는 매우 빠른 작업 전환을 통해 여러 작업이 동시에 진행되는 것처럼 보이게 만든다.

## 멀티 코어와 병렬 실행

CPU 코어 하나는 한 순간에 하나의 실행 흐름만 처리할 수 있다.
하지만 현대 CPU는 보통 여러 개의 코어를 가지고 있으므로 여러 실행 흐름을 실제로 동시에 처리할 수 있다.
이를 병렬 실행이라고 볼 수 있다.

현대 컴퓨터에서는 두 가지가 함께 일어난다.

- 운영체제는 시분할 기반 멀티 태스킹으로 여러 작업을 빠르게 전환한다.
- 멀티 코어 CPU는 여러 실행 흐름을 실제로 동시에 실행할 수 있다.

따라서 멀티 태스킹은 "동시에 실행되는 것처럼 보이게 하는 운영체제의 실행 방식"이고, 병렬 실행은 "여러 코어에서 실제로 동시에 실행되는 상태"라고 구분할 수 있다.

## 프로그램과 프로세스

프로그램은 실행되기 전 디스크에 저장된 파일이다.
아직 실행 중인 상태가 아니므로 CPU가 처리하는 대상은 아니다.

운영체제가 프로그램을 실행하기 위해 메모리에 올리고 실행에 필요한 자원을 할당하면 프로세스가 된다.
즉, 프로세스는 실행 중인 프로그램이다.

프로세스는 프로그램을 실행하기 위한 환경과 자원을 제공하는 컨테이너 역할을 한다.
프로세스 내부에는 코드 영역, 데이터 영역, 힙 영역, 스택 영역 등이 있다.
각 프로세스는 독립된 메모리 공간을 가지므로 기본적으로 다른 프로세스의 메모리에 직접 접근할 수 없다.

## 스레드

스레드는 프로세스 안에서 실제로 코드를 실행하는 흐름이다.
프로세스가 실행 환경과 자원을 제공한다면, 스레드는 실제 CPU를 사용해서 코드를 하나씩 실행한다.

하나의 프로세스는 최소 하나의 스레드를 가진다.
프로세스 안에는 여러 스레드가 존재할 수 있으며, 이 경우 하나의 프로세스 내부에 여러 실행 흐름이 생긴다.

같은 프로세스 안의 스레드들은 코드 영역, 데이터 영역, 힙 영역 같은 프로세스 자원을 공유한다.
반면 각 스레드는 자신만의 스택을 가진다.
스택에는 메서드 호출 정보와 지역 변수 등이 저장된다.

이 구조 때문에 멀티 스레드는 효율적일 수 있지만, 동시에 위험해질 수도 있다.
여러 스레드가 같은 힙 영역의 객체나 공유 상태에 접근할 수 있기 때문에 동시성 문제가 발생할 수 있다.

## 멀티 프로세스와 멀티 스레드

멀티 프로세스는 여러 프로세스를 사용하는 실행 구조다.
프로세스끼리는 기본적으로 메모리 공간이 분리되어 있으므로 안정성이 높다.
다만 프로세스 간 데이터를 주고받으려면 별도의 통신 방법이 필요하다.

멀티 스레드는 하나의 프로세스 안에서 여러 스레드를 실행하는 구조다.
스레드끼리는 같은 프로세스의 자원을 공유하므로 프로세스 간 통신보다 가볍게 협력할 수 있다.
하지만 공유 자원에 대한 접근을 잘 제어하지 않으면 race condition 같은 문제가 발생할 수 있다.

멀티 코어는 하드웨어 구조를 말하고, 멀티 프로세스는 여러 프로세스를 사용하는 실행 방식을 말한다.
따라서 멀티 코어를 이용해 동시에 처리하는 것 자체를 멀티 프로세스라고 표현하기보다는 병렬 처리 또는 병렬 실행이라고 표현하는 것이 더 정확하다.

## 스레드 스케줄링

운영체제는 실행 가능한 스레드들을 관리하고, 어떤 스레드를 언제 실행할지 결정한다.
이 과정을 스케줄링이라고 한다.

단순화해서 보면, 실행 가능한 스레드들은 스케줄링 큐에 들어간다.
운영체제의 스케줄러는 스케줄링 큐에서 실행할 스레드를 선택하고, CPU 코어는 선택된 스레드를 실행한다.
스레드는 일정 시간 동안 실행되다가 멈추고, 다시 실행 가능한 상태가 되면 큐에 들어갈 수 있다.
운영체제는 이 과정을 반복하면서 여러 스레드가 실행되도록 관리한다.

스케줄링은 단순히 시간을 동일하게 나누는 방식만 사용하지 않는다.
운영체제는 우선순위, 대기 시간, 입출력 상태, CPU 사용 패턴 등 여러 기준을 고려해 어떤 스레드를 실행할지 결정할 수 있다.
이런 방식들을 스케줄링 최적화 기법이라고 볼 수 있다.

CPU 코어가 하나라면 한 순간에 하나의 스레드만 실행된다.
하지만 스케줄러가 빠르게 실행 대상을 바꾸기 때문에 여러 스레드가 동시에 실행되는 것처럼 느껴진다.
CPU 코어가 여러 개라면 여러 스레드가 실제로 병렬로 실행될 수도 있다.

## 컨텍스트 스위칭

컨텍스트 스위칭은 실행 중인 스레드를 멈추고 다른 스레드로 전환하기 위해 필요한 작업이다.

CPU가 다른 스레드를 실행하려면 운영체제가 이전 스레드의 CPU 레지스터 상태와 실행 문맥을 저장하고, 다음 스레드의 실행 문맥을 복원해야 한다.
이 상태 저장과 복원 과정이 컨텍스트 스위칭이다.

컨텍스트 스위칭은 멀티 태스킹과 멀티 스레드 실행을 가능하게 하지만 비용이 발생한다.
따라서 스레드를 많이 만든다고 항상 성능이 좋아지는 것은 아니다.
스레드 수가 지나치게 많으면 실제 작업보다 스레드 전환 비용이 커질 수 있다.

## 사용자 스레드와 데몬 스레드

자바 스레드는 크게 사용자 스레드와 데몬 스레드로 구분할 수 있다.

사용자 스레드는 일반적으로 사용하는 보통의 스레드다.
JVM은 실행 중인 사용자 스레드가 하나라도 남아 있으면 계속 실행된다.
모든 사용자 스레드가 종료되면 JVM도 종료된다.

데몬 스레드는 사용자 스레드를 보조하는 백그라운드 스레드다.
데몬 스레드가 실행 중이더라도 모든 사용자 스레드가 종료되면 JVM은 종료된다.
따라서 데몬 스레드는 애플리케이션의 핵심 작업보다는 보조 작업에 사용해야 한다.

데몬 스레드는 일반 스레드와 동일하게 생성한 뒤 `setDaemon(true)`로 설정할 수 있다.
단, `setDaemon(true)`는 스레드를 시작하기 전, 즉 `start()`를 호출하기 전에 설정해야 한다.

## CPU 바운드와 I/O 바운드

스레드 수를 정할 때는 작업의 성격을 구분해야 한다.

CPU 바운드 작업은 CPU 연산을 많이 사용하는 작업이다.
계산 로직, 시뮬레이션, 대량 연산 처리 등이 여기에 가깝다.
CPU 바운드 작업은 CPU를 계속 사용하므로 스레드 수를 CPU 코어 수와 비슷하게 맞추는 것이 좋다.
일반적으로 CPU 코어 수 또는 CPU 코어 수보다 1개 정도 많은 수준에서 시작할 수 있다.

I/O 바운드 작업은 파일, 네트워크, 데이터베이스 같은 입출력 대기 시간이 많은 작업이다.
웹 애플리케이션은 요청 처리 중 데이터베이스나 외부 API 응답을 기다리는 시간이 많기 때문에 I/O 바운드 성격을 가지는 경우가 많다.
I/O 바운드 작업은 스레드가 대기하는 시간이 길기 때문에 CPU 코어 수보다 많은 스레드를 사용할 수 있다.

다만 적절한 스레드 수는 애플리케이션의 특성, I/O 대기 시간, 처리량 목표에 따라 달라진다.
따라서 정확한 값은 성능 테스트를 통해 찾아야 한다.

## 핵심 정리

- 프로그램은 실행되기 전 디스크에 있는 파일이다.
- 프로세스는 실행 중인 프로그램이며, 실행 환경과 자원을 제공하는 컨테이너 역할을 한다.
- 스레드는 프로세스 안에서 실제로 코드를 실행하는 흐름이다.
- 하나의 프로세스는 최소 하나의 스레드를 가진다.
- 프로세스끼리는 기본적으로 메모리가 분리되어 있다.
- 같은 프로세스의 스레드끼리는 자원을 공유하지만, 각자 스택은 따로 가진다.
- 멀티 태스킹은 운영체제가 여러 작업을 짧은 시간 단위로 번갈아 실행하는 방식이다.
- 멀티 코어 환경에서는 여러 스레드가 실제로 동시에 실행될 수도 있다.
- 운영체제는 스케줄링 큐와 스케줄링 기법을 통해 여러 스레드의 실행 순서를 관리한다.
- 컨텍스트 스위칭은 스레드 전환을 위해 실행 상태를 저장하고 복원하는 작업이며 비용이 발생한다.
- 사용자 스레드가 모두 종료되면 JVM은 종료되며, 데몬 스레드는 JVM 종료를 막지 않는다.
- CPU 바운드 작업은 CPU 코어 수와 비슷한 스레드 수에서 시작하고, I/O 바운드 작업은 코어 수보다 많은 스레드를 사용할 수 있다.
- 적절한 스레드 수는 성능 테스트를 통해 확인해야 한다.
