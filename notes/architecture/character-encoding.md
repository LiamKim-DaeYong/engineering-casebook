# 문자 인코딩

## 개요

컴퓨터는 모든 데이터를 0과 1로 표현한다.
문자도 예외가 아니므로, 사람이 사용하는 문자를 컴퓨터가 저장할 수 있는 숫자 체계로 바꿔야 한다.

이 문서는 문자 인코딩과 디코딩, ASCII, EUC-KR, 유니코드, UTF-8의 기본 개념을 정리한다.

## 문자 인코딩과 디코딩

문자를 컴퓨터가 저장할 수 있는 숫자 형태로 바꾸는 것을 인코딩이라고 한다.
반대로 저장된 숫자를 다시 문자로 해석하는 것을 디코딩이라고 한다.

예를 들어 어떤 문자 표에서 `A`를 특정 숫자에 대응시켜 저장했다면,
그 숫자를 다시 읽어서 `A`로 해석하는 과정이 디코딩이다.

즉, 컴퓨터가 문자를 다룬다는 것은 결국 문자와 숫자 사이의 대응 관계를 사용하는 일이라고 볼 수 있다.

## ASCII

초기에는 ASCII 체계를 사용해 문자를 표현했다.
ASCII는 7비트 기반 문자 집합이므로 표현 가능한 문자가 128개로 제한된다.

이 범위 안에는 영문 대소문자, 숫자, 제어 문자, 일부 특수 문자가 포함된다.
영문 환경에서는 충분했지만, 여러 언어를 함께 표현하기에는 범위가 너무 작았다.

실제 저장은 1바이트 단위로 이루어지는 경우가 많았지만,
표준 ASCII 자체는 7비트 문자 집합으로 이해하는 것이 정확하다.

## 한글 인코딩과 EUC-KR

한글을 표현하기 위해 여러 인코딩 방식이 사용되었고,
대표적으로 EUC-KR 같은 방식이 널리 쓰였다.

EUC-KR은 보통 한글 완성형 기반 인코딩으로 설명한다.
즉, 자주 사용하는 완성된 한글 글자를 코드에 직접 대응시키는 방식이다.

이 방식은 실용적이었지만 한계도 있었다.

- 모든 한글을 충분히 표현하기 어렵다.
- 여러 언어를 함께 처리하는 시스템에서는 언어별 인코딩 문제를 따로 고려해야 한다.
- 시스템마다 지원하는 문자 범위가 달라서 호환성 문제가 생길 수 있다.

즉, 한글을 처리할 수는 있었지만 다국어 환경까지 자연스럽게 포괄하기에는 부족했다.

## 유니코드

이런 문제를 해결하기 위해 유니코드가 등장했다.
유니코드는 여러 언어의 문자를 하나의 통합된 체계 안에서 표현하기 위한 문자 표준이다.

한글, 한자, 아랍어, 화살표, 수학 기호, 이모지 같은 다양한 문자를 함께 다룰 수 있다.
그래서 오늘날 다국어 지원이 필요한 시스템에서는 유니코드 기반 문자 체계를 주로 사용한다.

중요한 점은 유니코드가 곧바로 "몇 바이트"를 뜻하는 것은 아니라는 점이다.
유니코드는 문자에 번호를 부여하는 표준이고,
그 번호를 실제 메모리나 파일에 어떤 방식으로 저장할지는 별도의 인코딩 규칙이 담당한다.

## UTF-8, UTF-16, UTF-32

유니코드를 저장하는 대표적인 방식으로는 UTF-8, UTF-16, UTF-32가 있다.

- UTF-8: 1바이트에서 4바이트까지 사용하는 가변 길이 인코딩
- UTF-16: 보통 2바이트 또는 4바이트를 사용하는 방식
- UTF-32: 고정 4바이트 방식

즉, "유니코드는 4바이트다"라고 이해하면 정확하지 않다.
4바이트를 사용하는 것은 UTF-32 같은 특정 인코딩 방식의 특징이다.

## UTF-8이 많이 쓰이는 이유

UTF-8은 오늘날 가장 널리 사용하는 유니코드 인코딩 방식이다.

이유는 다음과 같다.

- ASCII와 호환된다.
- 영문은 1바이트로 표현할 수 있다.
- 한글, 이모지, 여러 언어 문자를 함께 표현할 수 있다.
- 웹과 다양한 시스템 환경에서 기본처럼 사용된다.

즉, UTF-8은 표현 범위와 호환성의 균형이 좋아서 사실상 표준처럼 자리 잡았다.

## 핵심 정리

- 컴퓨터는 문자를 직접 저장하지 않고 숫자로 바꿔 저장한다.
- 문자를 숫자로 바꾸는 것은 인코딩, 다시 문자로 해석하는 것은 디코딩이다.
- ASCII는 7비트 기반 문자 집합이라 표현 가능한 문자가 제한적이다.
- EUC-KR은 한글 표현을 지원했지만 전체 문자 범위와 다국어 처리에 한계가 있었다.
- 유니코드는 여러 언어 문자를 하나의 체계로 다루기 위한 문자 표준이다.
- UTF-8, UTF-16, UTF-32는 유니코드를 저장하는 방식이다.
- UTF-8은 가변 길이 인코딩이며, 호환성과 범용성 때문에 가장 널리 사용된다.
